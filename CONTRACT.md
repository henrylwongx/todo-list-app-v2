# API Contract for Todo List Application

## Base URL
All endpoints are under `/api`.

## Todo Object
```json
{
  "id": integer,
  "title": string,
  "completed": boolean
}
```

## Endpoints

### GET /api/todos
Returns a list of all todos.

**Request:** None

**Response:**
- Status: 200 OK
- Body: Array of Todo objects

### POST /api/todos
Creates a new todo.

**Request:**
- Body: 
  ```json
  {
    "title": string (required)
  }
  ```
  Note: `completed` defaults to false, `id` is generated by the server.

**Response:**
- Status: 201 Created
- Body: The created Todo object (with generated `id` and default `completed: false`)

**Errors:**
- 400 Bad Request if `title` is missing or empty.

### DELETE /api/todos/{id}
Deletes a todo by its ID.

**Request:**
- Path parameter: `id` (integer)

**Response:**
- Status: 204 No Content (empty body)

**Errors:**
- 404 Not Found if no todo with the given `id` exists.

### PATCH /api/todos/{id}
Toggles the completion status of a todo.

**Request:**
- Path parameter: `id` (integer)
- Body: None (the action is to toggle)

**Response:**
- Status: 200 OK
- Body: The updated Todo object (with toggled `completed` field)

**Errors:**
- 404 Not Found if no todo with the given `id` exists.

## Notes
- The backend uses in-memory storage (a list) for simplicity.
- IDs are integers and are unique and incrementing.
- All JSON bodies must be sent with Content-Type: application/json.
